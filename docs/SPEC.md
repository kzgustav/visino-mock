## 1. プロジェクト概要

### 背景
恵比寿の一軒家サロン「visino ebisu」の既存ウェブサイト（http://visino.jp/）が約10年前に制作されたまま放置されており、デザインの老朽化・情報の矛盾・技術的な問題を多数抱えている。オーナーからの相談を受け、ゼロからのリニューアルを提案。

### 既存サイトの問題点
- **構造:** サイトタイトルが「ＪＲ恵比寿駅（東口）徒歩８分。」（サロン名ではない）
- **コンテンツ:** TOPページ（/concept）が空、ページ間で電話番号・定休日が矛盾
- **技術:** SSL未対応（HTTP）、WordPress無料テーマ放置、Copyright 2021で停止
- **SEO/GEO:** meta情報未設定、構造化データなし
- **デザイン:** サロンの魅力（一軒家、看板犬、¥4,000の価格設定）が伝わらない

### ゴール
- モダンで温かみのあるサイトに刷新
- オーナーが自分で更新できる簡易管理画面を提供
- GEO（AI検索最適化）対応
- ランニングコスト最小化（Firebase無料枠活用）
- 他の小規模店舗にも横展開できるパッケージの原型を構築

## 3. デザイン仕様

### デザインコンセプト
「温かみと品のバランス」— 高級すぎず、カジュアルすぎず、恵比寿の一軒家サロンという特別な空間を感じさせるデザイン。

### カラーパレット

| 変数名 | カラーコード | 用途 |
|--------|-------------|------|
| `--warm-cream` | `#FAF6F1` | 背景メイン |
| `--soft-beige` | `#F0E8DD` | 背景サブ、カード背景 |
| `--warm-taupe` | `#C4B5A4` | ボーダー、区切り線 |
| `--deep-brown` | `#5C4A3A` | ボタン、アイコン |
| `--rich-brown` | `#3D2E22` | テキストメイン、フッター |
| `--accent-rose` | `#C08B7E` | アクセント、ラベル |
| `--accent-rose-light` | `#D4A99A` | アクセント（明） |
| `--soft-sage` | `#A8B5A0` | 看板犬セクション等 |

### フォント

| 用途 | フォント | ウェイト |
|------|---------|---------|
| 英語見出し・ロゴ | Cormorant Garamond | 300, 400, 500 |
| 日本語見出し | Noto Serif JP | 200, 300, 400 |
| 本文 | Zen Kaku Gothic New | 300, 400, 500 |

### ブランド要素
- **ロゴ:** visinoオリジナルロゴ（顔のシルエット＋星3つ）をヒーロー背景に透過配置
  - PC: テキスト背後に大きく（opacity 0.12〜0.17、呼吸アニメーション）
  - モバイル: ボタンと写真の間に独立配置（opacity 0.35〜0.42）
  - ロゴは透過PNG化処理済み（元ファイルはJPEG黒背景 → Python PILで透過変換、ブラウンカラー#5C4A3Aで着色）
- **看板犬バッジ:** ヒーロー写真左下にフロートアニメーション付きバッジ

### レスポンシブ対応
- PC: 2カラムグリッドレイアウト
- モバイル（768px以下）: 1カラム、ハンバーガーメニュー、写真エリア全幅表示

### アニメーション
- スクロールリビール（IntersectionObserver）
- ナビバーのスクロール時シャドウ
- FAQアコーディオン開閉
- ロゴ呼吸アニメーション（opacity変動）
- 看板犬バッジのフロート

### 構造化データ（JSON-LD）

| スキーマ | 内容 |

### コンテンツ設計
- 「なぜ¥4,000なのか」セクション → AIが料金に関する質問に引用しやすい構造

- 施術の流れ → ステップ形式でAIが手順を正確に説明可能

## 5. 管理画面仕様

### 編集可能項目

| タブ | 項目 | フィールド |
|------|------|-----------|
| **お知らせ** | ニュース一覧 | 日付、タグ（お知らせ/重要）、テキスト |
| **基本情報** | サロン情報 | 郵便番号、住所、アクセス補足、目印、営業時間（平日/土日祝）、定休日、電話番号×2、LINE ID、Instagram |
| **小顔矯正** | 料金設定 | 施術料金、税表記、施術時間、ブライダル料金・税・回数・期限、紹介割引額、キャンセル料、定休日 |
| **スタッフ** | スタッフ一覧 | 名前（英語）、役職、紹介文 |
| **FAQ** | Q&A一覧 | 質問、回答 |

### 管理画面の機能
- タブ切り替え式UI
- リピーター（お知らせ・スタッフ・FAQ）: 追加・削除・編集
- 保存ボタン → localStorage保存（モック段階）
- JSONエクスポート → site-data.jsonダウンロード（バックアップ用）
- トースト通知

### データフロー（現行モック）
```
admin.html → 編集 → localStorage保存
index.html → localStorage or site-data.json読み込み → 動的描画
```

### データフロー（本番・Firebase移行後）
```
admin.html → 編集 → Firestore保存（即時反映）
index.html → Firestore読み込み → 動的描画
```

## 9. パッケージ構想

### コンセプト
「AIと人間のパートナーシップで、プロ品質のサイトを低コスト・低メンテで提供する」

### パッケージ内容（想定）

| 項目 | 内容 |
|------|------|
| **初期費用** | ¥30,000〜50,000 |
| **含まれるもの** | ヒアリング、デザイン、コーディング、Firebase設定、ドメイン設定、管理画面構築、使い方レクチャー |
| **ランニング** | ドメイン実費のみ or 年¥10,000でサポート付き |
| **管理画面** | お知らせ、料金、営業時間、スタッフ、FAQ、写真の更新 |

### 想定ターゲット
- 個人サロン（美容、ネイル、整体、鍼灸）
- 小規模飲食店
- 個人教室・スクール
- その他、現在のサイトに不満を持つ小規模事業者

### 競合優位性
- WordPress制作会社（20〜50万）と比較して圧倒的に安い
- Wix/STUDIOの「自分で頑張る」モデルと違い、プロ品質で納品
- メンテナンスコストほぼゼロ（Firebase無料枠、WordPress的な脆弱性リスクなし）
- GEO対応（AI検索への最適化は従来の制作会社がほぼ対応していない領域）
